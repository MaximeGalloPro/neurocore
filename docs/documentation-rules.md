# Regles de documentation

Chaque sujet significatif du projet doit repondre autant que possible aux questions suivantes.

## Questions obligatoires

### Pourquoi ?

Expliquer le probleme, le besoin ou l'hypothese.

### Comment ?

Decrire la methode, l'architecture ou le protocole.

### Quelles alternatives existent ?

Lister les options raisonnables, meme si elles ne sont pas retenues.

### Quels sont les avantages ?

Expliquer ce que la solution apporte.

### Quels sont les inconvenients ?

Documenter les limites, couts, risques et inconnues.

### Comment le reproduire ?

Donner les etapes necessaires pour refaire le travail.

### Quelles publications scientifiques le decrivent ?

Citer les sources, publications, notes techniques ou ressources utiles.

## Regles de style

- Ecrire comme un ingenieur.
- Separer les faits, hypotheses et decisions.
- Documenter les echecs sans les maquiller.
- Preferer une note courte publiee rapidement a une note parfaite jamais terminee.
- Ajouter une date quand une decision ou une observation depend du contexte.
